### Key Decisions
- D-001 Catalog sync is a NEW layer next to existing ProviderCatalogRefresher,
  not a replacement. Refresher = vendor doc scraping. Sync = curated catalog.
- D-002 Public repo (fusionaize-metadata-public) holds providers/ + schemas/.
  Private repo (fusionaize-metadata) keeps products/gate/overlays.v1.json.
- D-003 Auth via fine-grained PAT with FAIGATE_METADATA_TOKEN env var.
  No OAuth flow in v1 — too heavy.
- D-004 Schema bumps additively to v1.1 with expires_at + tier_status.
  Old faigate versions ignore the new fields.
- D-005 Local FAIGATE_PROVIDER_METADATA_DIR override still wins over remote
  sync — preserves dev workflow.
